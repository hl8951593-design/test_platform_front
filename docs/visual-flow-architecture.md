# 可视化编排模块功能逻辑说明

## 1. 模块定位

可视化编排模块入口为 `/flow`。它负责把当前项目中的 HTTP/WebSocket 测试用例与开始、结束、条件、等待等控制节点组合成有向无环图（DAG），并配置节点之间的执行顺序和数据传递关系。

相关代码：

| 文件 | 职责 |
| --- | --- |
| `src/pages/FlowPage.tsx` | 页面状态、资产列表、画布拖拽、连线、属性配置、导入导出、保存与试运行 |
| `src/api/flows.ts` | 数据类型、流程校验、流程接口封装 |
| `src/styles.css` | 编排页面样式 |
| `src/pages/FlowPage.test.tsx` | 页面交互测试 |
| `src/api/flows.test.ts` | 流程结构与执行前校验测试 |
| `api_docs/api_visual_flows.md` | 后端接口、数据结构和执行器契约 |

## 2. 核心概念

| 概念 | 含义 |
| --- | --- |
| 流程定义 | 一个可保存、导入、导出、试运行的完整编排图 |
| 节点 | 测试用例或流程控制单元 |
| 连线 | 两个节点之间的执行依赖和路由条件 |
| 上游节点 | 通过连线直接指向当前节点的节点 |
| 下游节点 | 当前节点通过连线直接指向的节点 |
| 输入取值关联 | 把直接上游节点的输出写入当前节点请求字段 |
| 可引用输出 | 当前节点声明给后续节点使用的标准化输出路径 |

连线和输入取值关联必须分开理解：

- `edges` 决定执行依赖、执行方向和路由条件。
- `inputBindings` 决定运行时数据如何从上游传递到当前节点。

## 3. 页面功能逻辑

### 3.1 项目与环境切换

进入页面或切换项目时：

1. 清空当前画布、选中节点和连线状态。
2. 并行加载 HTTP 测试用例、WebSocket 测试用例和已保存流程列表。
3. 单个资产接口失败不会阻塞其他资产展示。
4. 保存和试运行时始终使用顶部当前项目和当前环境。

切换环境时保留当前画布，只更新流程默认环境和后续试运行环境，避免清空未保存编排。

### 3.2 左侧资产区

- 测试用例来自当前项目的 `/test-cases` 和 `/websocket-test-cases`。
- 支持按用例名称、请求方法和路径搜索。
- 可把测试用例或控制组件拖入画布。
- 双击资产也可快速新增节点。
- 点击已保存流程后加载完整流程定义。

### 3.3 画布操作

- 节点可拖动，位置保存为 `position.x`、`position.y`。
- 画布支持 `60%` 至 `140%` 缩放。
- 输出端口拖到目标输入端口可建立连线。
- 也支持先点击输出端口，再点击输入端口建立连线。
- 禁止自连接、重复连接和形成环。
- 点击连线会删除连线，并同步删除目标节点中依赖该上游节点的输入取值关联。
- 删除节点会删除其所有入边、出边，并清理其他节点对它的输入取值引用。
- 复制节点会生成新的节点 ID 和新的绑定 ID，不复制连线。

### 3.4 连线路由

每条连线必须包含 `route`：

| route | 执行条件 |
| --- | --- |
| `always` | 上游完成后始终进入下游 |
| `success` | 上游执行成功时进入下游 |
| `failure` | 上游执行失败时进入下游 |
| `true` | 条件节点表达式结果为 true 时进入下游 |
| `false` | 条件节点表达式结果为 false 时进入下游 |

新增普通节点连线时默认使用 `always`。条件节点新增第一条出边默认使用 `true`，第二条默认使用 `false`。用户可在右侧“连接关系”的下游节点列表中修改路由。

### 3.5 右侧属性配置

选中节点后展示：

- 节点名称和说明。
- 节点失败后是否继续执行。
- 条件节点表达式。
- 等待节点等待毫秒数。
- 直接上游节点名称、类型和方法。
- 直接下游节点名称、类型、方法和连线路由。
- 输入取值关联。
- 可引用输出路径。
- HTTP/WebSocket 节点本地 `caseConfig` 和 `caseOverrides`。

点击连接关系中的节点名称，可切换到对应节点。

### 3.6 输入取值关联

输入取值关联只能引用当前节点的直接上游节点。

```json
{
  "id": "binding-token",
  "target": "headers.Authorization",
  "sourceNodeId": "node-login",
  "sourcePath": "response.body.data.token",
  "fallback": ""
}
```

字段含义：

| 字段 | 说明 |
| --- | --- |
| `target` | 写入当前节点请求配置的目标路径 |
| `sourceNodeId` | 数据来源节点 ID，必须是直接上游 |
| `sourcePath` | 从来源节点标准化输出中读取的路径 |
| `fallback` | 来源路径不存在时使用的可选默认值 |

建议支持的 `target` 根路径：

- `pathParams`
- `query`
- `headers`
- `body`
- `variables`
- `messages`（WebSocket）

`sourcePath` 必须等于来源节点声明的某个 `outputPaths`，或位于该声明路径之下。例如来源节点声明 `response.body` 后，可引用 `response.body.data.id`。

### 3.7 节点本地用例配置

HTTP 和 WebSocket 节点支持：

- `config.caseConfig`：当前节点使用的完整本地用例配置。
- `config.caseOverrides`：在本地配置基础上再次覆盖的字段。

执行应用顺序：

1. 深拷贝引用测试用例。
2. 应用 `caseConfig`。
3. 应用 `caseOverrides`。
4. 应用 `inputBindings`。
5. 作为临时未保存用例执行。

本地配置只保存在流程定义中，不修改引用测试用例，也不更新引用用例的最近执行时间和状态。项目归属、测试用例身份和执行环境不能被节点本地配置覆盖。

属性区使用 JSON 对象编辑器维护这两个字段。存在 JSON 格式错误时禁止保存和试运行。

### 3.8 导入与导出

- 导出文件为完整 `FlowDefinition` JSON。
- 导入支持完整定义，或 `{ "definition": FlowDefinition }` 包装格式。
- 导入后强制使用当前项目 ID。
- 导入流程会清除 `id` 和 `updatedAt`，保存时按新流程创建，禁止覆盖原流程。
- 导入时补齐旧数据缺少的 `route`、`inputBindings`、`outputPaths` 和 `viewport.zoom`。
- 导入后执行结构校验；存在悬空连线、重复 ID、循环或无效绑定时拒绝导入。

### 3.9 保存与试运行

- 保存对当前定义执行结构校验，已存在 `id` 时更新流程，否则创建流程。
- 编排页“试运行”始终提交当前画布完整定义到 `/api/v1/flows/execute-unsaved`。
- 每次执行请求发送唯一 `Idempotency-Key`。
- 当前后端同步执行，前端等待响应后展示执行编号。
- 即使当前流程已经保存，画布可能包含未保存修改，因此编排页不能直接按流程 ID 执行旧版本。
- `/api/v1/flows/{flow_id}/execute` 供执行中心、测试计划或明确执行已保存版本的场景使用。

## 4. 校验规则

### 4.1 保存前结构校验

保存草稿时允许流程暂时没有开始或结束节点，但必须满足：

- 节点 ID 和连线 ID 非空且唯一。
- 节点名称非空。
- 接口节点必须引用真实测试用例。
- 条件节点必须有表达式。
- 等待时间必须为非负有限数字。
- 连线的 source/target 必须存在。
- 禁止自连接、重复连接和循环。
- 输入取值关联必须填写完整。
- 输入取值来源必须是直接上游节点。

### 4.2 试运行前可执行校验

试运行除满足结构校验外，还必须满足：

- 必须且只能有一个开始节点。
- 至少有一个结束节点。
- 开始节点不能有上游。
- 结束节点不能有下游。
- 所有节点必须能从开始节点到达。
- 所有节点必须能到达某个结束节点。
- 普通节点不能使用 `true`/`false` 路由。
- 每个条件节点必须且只能有一条 `true` 出边和一条 `false` 出边。

后端保存接口必须重复执行结构校验；执行接口必须重复执行完整可执行校验，不能只依赖前端。

## 5. 状态一致性规则

- 删除连线：同步删除目标节点中 `sourceNodeId` 等于该连线源节点的绑定。
- 删除节点：同步删除其所有连线和所有其他节点对它的绑定。
- 复制节点：复制属性，不复制连线；节点 ID 和绑定 ID 全部重新生成。
- 修改节点名称：不影响引用关系，引用关系只使用节点 ID。
- 修改测试用例原始资产：已保存流程仍保存引用 ID、显示名称、方法和路径快照；执行时后端应按引用 ID 获取最新用例定义。
- 保存成功但流程列表刷新失败：保存仍视为成功，列表刷新失败不应误报保存失败。

## 6. 当前前端与后端责任边界

前端负责：

- 提供编辑体验和即时校验。
- 维护画布位置、缩放、连线和属性状态。
- 提交完整流程定义。
- 展示后端错误信息。

后端负责：

- 项目权限和引用资产权限校验。
- 流程结构与可执行校验。
- 版本管理和并发更新控制。
- 运行时变量解析、节点调度、重试、超时和结果持久化。
- 敏感数据脱敏。
- 保证执行使用不可变流程版本快照。

## 7. 当前实现范围

前端已经实现：

- 当前项目 HTTP/WebSocket 用例资产加载和搜索。
- 节点拖入、双击新增、拖动、复制、删除和自动布局。
- 画布缩放和自适应中间区域。
- 点击连线、拖拽连线、连线预览、循环连接阻止。
- 上下游节点名称和路由属性展示。
- 输入取值关联和可引用输出配置。
- 导入、导出、保存和当前画布试运行。
- 结构校验与可执行校验。

前端形成完整闭环所依赖的后端能力，当前后端已经提供：

- 流程列表、详情、创建、更新和版本管理。
- 流程实际执行、节点状态、执行日志和执行结果。
- 条件表达式安全解析。
- HTTP/WebSocket 用例执行快照。
- 节点并行调度、分支汇合、取消、超时和并发限制。
- 权限、幂等、乐观锁和敏感数据脱敏。

当前页面尚未提供流程删除、版本对比、执行结果实时展示、撤销/重做和小地图。这些能力不影响当前后端流程定义与执行器契约，可后续扩展。

## 8. 后端联调状态

当前后端已提供：

- 流程列表、创建、详情、更新。
- 已保存流程和未保存流程同步执行。
- 版本化 DAG 持久化、不可变版本和 SHA-256 定义哈希。
- 流程与节点执行快照。
- 项目引用、绑定、路由、循环、开始/结束、可达性和条件路由校验。
- `Idempotency-Key` 执行幂等。
- 受限 Python/CEL 风格条件表达式。
- 执行上下文、请求和输出快照脱敏。
- 节点本地 `caseConfig`、`caseOverrides` 临时用例执行。

当前后端执行为同步模式。Celery 调度、执行取消和项目并发限制仍属于后续生产增强能力。
